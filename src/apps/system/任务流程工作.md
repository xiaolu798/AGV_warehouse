第一阶段：建立 TCP 握手
首先，你的后端必须有一个运行中的 TCP Server（通常在 FastAPI 启动时异步开启），监听某个端口（比如 8080）。

启动后端服务：确保你的 C# 或 Python 写的 TCP 服务器已经跑起来了。

网络调试助手配置：

协议类型：选择 TCP Client。

远程主机地址：输入 127.0.0.1。

远程主机端口：输入你的后端监听端口（如图中所示的 8080）。

点击连接：当中间的红色指示灯变亮，说明模拟设备（调试助手）已经成功“上线”。

第二阶段：任务触发与信息下发
由于你已经写好了 /warehouse 接口来初始化库位矩阵，现在的逻辑流应该是：

网页下单：你在前端大屏上选好起点和终点，点击创建任务。

后端处理：后端保存任务到 wms_mission 表，并将状态设为 0（待处理）。

发送指令到助手：

后端会通过 TCP 连接找到你的调试助手，并发送一条数据。

你会看到：调试助手的“数据日志”窗口弹出一条信息，例如：{"mission_id": 101, "from": "A-01-01", "to": "B-05-05"}。

第三阶段：模拟硬件反馈（关键操作）
当你在调试助手里看到指令后，你需要模拟 AGV 搬运完成并告诉服务器。

准备信息内容：
在助手的“数据发送”框里输入你的反馈协议。通常为了简单，我们会发送 JSON 字符串，例如：
{"type": "FINISH", "mission_no": "MS20260419001", "hardware_id": "001"}

点击发送：

后端 TCP Server 收到这条消息。

后端解析出 mission_no，然后调用数据库将该任务状态更新为 2（已完成）。

自动变色：因为你在 get_task_list 里已经处理好了 MissionOutSchema 的关联数据，WebSocket 会立刻通知前端，你大屏上的库位格子会从橙色变为蓝色/灰色。
